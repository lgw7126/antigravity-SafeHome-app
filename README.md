## 🎯 1. 기획 의도 (Planning Intention)

### 💡 탄생 배경 및 필요성
기존의 택시 호출 앱(카카오 T, UT 등)은 스마트폰 조작과 결제 카드 등록이 서툰 시니어(노년층) 세대에게는 너무나도 높은 진입 장벽입니다. 
- 복잡한 지도 화면에서 출발지와 목적지를 검색하고 입력하는 과정의 어려움
- 기기 내 결제 수단 등록 및 인증 실패에 따른 당혹감
- GPS 수신 불량 혹은 권한 설정 오류 시 적절한 대처의 한계

### 🌟 핵심 가치
**"부모님은 원터치 터치 한 번으로 안전하게 위치를 전송하고, 자녀는 관제 대시보드를 보며 대신 택시를 호출합니다."**
본 서비스는 시니어 사용자의 디지털 페인 포인트(Pain Point)를 해결하기 위해 설계된 **대리 호출 브릿지 플랫폼**입니다. 부모와 자녀 간의 디지털 능력을 연결(Bridge)하여 시니어층의 안전하고 편리한 귀가를 돕습니다.

### 🔌 기술적 차별점
- **무입력(Zero-Typing) 경험**: 부모님 화면은 텍스트 입력을 전혀 요구하지 않고 거대한 버튼 터치와 음성 연결로만 구성되어 있습니다.
- **서버리스 실시간 브릿지 통신**: 별도의 데이터베이스나 복잡한 백엔드 구축 없이, 공용 SSE(Server-Sent Events) 브로커인 `ntfy.sh`를 활용해 서로 다른 기기(부모님 스마트폰 ↔️ 자녀 PC/스마트폰) 간 실시간 위치 데이터 및 목적지 설정을 동기화합니다.
- **API 유연성 및 자율 구동 (Graceful Degradation)**: 카카오맵 API Key가 미설정된 환경에서도 오픈 지도 API(Nominatim)를 폴백으로 제공하여 주소 역변환 및 통신 흐름이 정상 작동하도록 안정성을 기했습니다.

## 🛠️ 2. 주요 기능 및 디자인 명세

### ① 부모님용 화면 (👵 부모 모드)
- **대형 원터치 위치 전송**: 조작이 편리한 대형 버튼과 확실한 시각 피드백으로 위치 권한 모달 동의 후 즉시 자녀에게 위치를 발송합니다.
- **목적지 퀵 전송 버튼**: 미리 설정된 5가지 퀵 목적지(집, 복지관, 딸집, 아들집, 병원 등)를 터치 한 번으로 골라 보낼 수 있습니다.
- **상시 비상 전화 버튼**: 화면 하단에 `📞 자녀에게 전화해서 위치를 알려주세요` 버튼을 상시 배치하여, 위급하거나 설명이 어려운 상황에 언제든지 목소리로 연결할 수 있도록 유도합니다.
- **위치 권한 전처리 안내**: 위치 수집 동의 전에 불안감을 덜 수 있도록 부드러운 일러스트 느낌의 안내 모달을 노출합니다.
- **기지국 IP 기반 위치 폴백**: GPS 수신이 불가능할 경우 IP 기반 위치를 수집하여 자녀에게 발송하고 직통 비상 전화 단추를 활성화합니다.

### ② 자녀용 화면 (🧑‍💻 자녀 관제 모드)
- **실시간 지도 관제 및 주소 변환**: 부모님이 전송한 위경도 좌표를 실시간으로 지도 핀에 매핑하고, 도로명/지번 한글 상세 주소로 한눈에 보여줍니다.
- **3화음 알림 및 OS 푸시 알림**: 호출 발생 시 귀가 안심 3화음 벨소리가 재생되고, 브라우저가 백그라운드에 있을 때도 데스크톱 알림이 전송됩니다.
- **원터치 주소 복사 및 앱 연동**: 출발지와 도착지 주소를 클릭 한 번으로 복사하고, 모바일 카카오 T 앱 딥링크 스킴(`kakaot://`) 및 카카오맵 길찾기 웹 링크를 연동하여 자녀가 즉각적으로 대리 호출과 길안내를 수행할 수 있게 돕습니다.
- **원격 목적지 동기화 설정**: 자녀 화면에서 부모님의 퀵 목적지 리스트를 수정한 뒤 동기화 버튼을 누르면, 부모님 화면의 버튼들이 실시간으로 무선 자동 업데이트됩니다.

### ③ 데모 듀얼 뷰 (📱↔️💻 Dual View)
- 한 화면 안에서 **[왼쪽: 부모용 기기 | 오른쪽: 자녀용 기기]** 형태로 분할 작동하는 시뮬레이터를 제공하여, 단일 브라우저 탭에서도 양방향 실시간 무선 연동 흐름을 직관적으로 확인하고 시연할 수 있습니다. (발표 및 데모 시 유용)

## 📖 3. 사용 방법 및 흐름

### ⚙️ 최초 1회 연동 설정 방법
1. 앱 우측 상단의 **톱니바퀴(⚙️) 아이콘**을 클릭해 설정 창을 엽니다.
2. **카카오맵 API 등록 (선택)**: [Kakao Developers](https://developers.kakao.com/) 에서 발급받은 `JavaScript 키`를 입력하면 지도가 활성화됩니다. *(미입력 시에도 텍스트 주소는 정상 작동합니다)*
3. **자녀 연락처 설정**: 부모용 화면의 전화 걸기 버튼과 연동할 자녀의 전화번호를 입력합니다.
4. **브릿지 연동 토픽 ID**: 부모와 자녀 간 통신 채널을 맞춰주기 위해 고유한 토픽 ID(예: `bridge-mobility-1234`)를 설정합니다. **부모용 기기와 자녀용 기기의 설정 창에 동일한 토픽 ID가 입력되어야 무선으로 실시간 연동됩니다.**
5. **자주 가는 목적지 설정**: 집, 병원, 복지관 등 자주 가는 목적지 주소와 노출될 버튼명을 저장합니다.

### 👵 부모님 이용 흐름
1. **위치만 보내기**: 메인 화면의 큰 **[📢 위치 보내기]** 버튼을 터치합니다. 위치 수집 완료 후 안내 팝업이 뜨면 **[이전 화면으로]**를 눌러 복귀합니다.
2. **가고 싶은 곳과 함께 보내기**: 화면 중앙의 **[🏠 집으로]**, **[🏥 병원으로]** 등 설정된 5개의 버튼 중 하나를 터치하면 위치와 함께 행선지가 자녀에게 전송됩니다.
3. **전화하며 알려주기**: 하단의 초록색 **[📞 자녀에게 전화해서 위치를 알려주세요]** 대형 버튼을 눌러 자녀와 직접 음성 통화를 연결합니다.

### 🧑‍💻 자녀 이용 흐름
1. 관제 대시보드 화면을 열고 부모님의 호출을 대기합니다.
2. 부모님이 위치를 발송하면 **알림음**과 함께 **데스크톱 브라우저 팝업**이 뜹니다.
3. 수신된 출발지 주소와 도착지 주소 우측의 **[📋 복사]** 버튼을 눌러 주소를 복사합니다.
4. **[💬 카카오 T 호출앱 열기]** 또는 **[🗺️ 카카오맵]** 버튼을 클릭하여 복사한 주소로 신속하게 대리 배차 및 경로 확인을 진행합니다.
5. 부모님의 퀵 목적지 버튼을 실시간으로 변경해주고 싶다면, 설정창이나 우측 사이드바 패널에서 수정 후 **[변경사항 부모님께 동기화]**를 클릭합니다.

## 🌐 4. GitHub Pages 배포 및 실행 방법

이 앱은 서버가 필요 없는 정적 웹 애플리케이션으로, 깃허브에 업로드한 후 **GitHub Pages** 기능을 활성화하면 전 세계 어디서나 웹 링크 하나로 접속해 바로 실행할 수 있습니다.

### 1단계: 깃허브에 폴더 업로드
이 프로젝트 폴더 전체를 본인의 깃허브 저장소(Repository)에 커밋 및 푸시(Push)합니다.

### 2단계: GitHub Pages 활성화
1. 업로드한 깃허브 저장소 페이지 상단의 **Settings** 메뉴로 이동합니다.
2. 좌측 사이드바에서 **Pages**를 클릭합니다.
3. **Build and deployment** 섹션의 **Source**에서 `Deploy from a branch`를 선택합니다.
4. **Branch** 설정을 `main` (또는 `master`) 브랜치와 `/ (root)` 폴더로 설정한 후 **Save** 버튼을 누릅니다.
5. 약 1~2분 후 페이지 상단에 배포 완료 안내와 함께 생성된 웹 실행 링크가 표시됩니다.
   - 본 프로젝트의 Pages 실행 주소: [https://lgw7126.github.io/antigravity-SafeHome-app/](https://lgw7126.github.io/antigravity-SafeHome-app/)
